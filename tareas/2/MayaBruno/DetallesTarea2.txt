El problema abordado fue la comparación de estrategias de planificación de procesos en sistemas operativos, el cual consiste en determinar la secuencia óptima de ejecución de un conjunto de procesos para minimizar métricas clave de rendimiento, como el tiempo de retorno (T), el tiempo de espera (E) y el factor de penalización (P).El programa fue desarrollado en el lenguaje Python, utilizando el entorno de desarrollo de sublime text.
Para ejecutar la simulación, se necesita tener instalado Python y abrir una terminal en la carpeta que contiene el archivo. Posteriormente, se debe ejecutar con el comando: Python tarea2bmt.py.
La estrategia de simulación empleada consistió en implementar y ejecutar cuatro algoritmos de planificación FCFS (First-Come, First-Served), SPN (Shortest Process Next), RR1 (Round Robin con Quantum 1) y RR4 (Round Robin con Quantum 4)en cinco conjuntos de procesos predefinidos. Esto permitió analizar el comportamiento de cada algoritmo.
